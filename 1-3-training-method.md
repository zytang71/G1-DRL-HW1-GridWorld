# 1-3 訓練流程規劃（在 1-2 的基礎上）

## 目標
- 以 `1-2` 產生的隨機策略與地圖為起點。
- 加入獎勵函數後，透過反覆迭代直到收斂，完成實際訓練。
- 輸出收斂後的狀態價值 `V(s)` 與更新後策略。

## 前置條件（承接 1-2）
- 已完成地圖設定：`n`、起點、終點、`n-2` 障礙物。
- 已有每個非終止狀態的初始策略（隨機箭頭）。

## 獎勵設計
- 一般步驟獎勵：`step_reward = -1`
- 到達終點獎勵：`goal_reward = +10`
- 折扣係數：`gamma = 0.9`（可調）

## 狀態轉移規則
- 動作集合：`A = {Up, Down, Left, Right}`。
- 若動作導致越界或撞到障礙物，則維持原位（`s' = s`）。
- 終點為終止狀態，不再展開後續轉移。

## 訓練方法（建議：Policy Iteration）
1. **初始化策略**
   - 使用 1-2 的隨機策略作為 `pi_0`。
2. **策略評估（Policy Evaluation）**
   - 固定當前策略 `pi`，反覆更新每個狀態價值：
   - `V_{k+1}(s) = r(s,pi(s),s') + gamma * V_k(s')`
   - 直到 `max_s |V_{k+1}(s)-V_k(s)| < theta`（例如 `theta = 1e-4`）。
3. **策略改進（Policy Improvement）**
   - 對每個狀態選擇使下式最大的動作：
   - `argmax_a [ r(s,a,s') + gamma * V(s') ]`
   - 形成新策略 `pi'`。
4. **重複 2~3**
   - 若策略不再改變（stable）或價值變化低於門檻，即視為收斂。

## 收斂與停止條件
- 評估收斂門檻：`theta`（例如 `1e-4`）。
- 最多迭代次數：`max_iterations`（例如 1000）避免無限迴圈。
- 策略穩定（前後兩次策略完全一致）可提早停止。

## 輸出結果
- 每個可通行狀態的 `V(s)`。
- 收斂後策略（箭頭）。
- 訓練統計資訊：
  - 迭代回合數
  - 最終 `delta`
  - 是否收斂

## 頁面呈現建議
- 先顯示 1-2 的隨機策略箭頭。
- 點擊「開始訓練」後顯示：
  - 每格 `V(s)`（小數 2~3 位）
  - 更新後策略箭頭
  - 訓練狀態（Converged / Not Converged）

## API 建議
- `POST /api/generate-policy`：輸入地圖，回傳隨機策略。
- `POST /api/train-policy`：輸入地圖、初始策略、`step_reward`、`goal_reward`、`gamma`、`theta`，回傳收斂結果。
